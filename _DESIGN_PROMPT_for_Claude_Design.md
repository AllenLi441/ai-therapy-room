# Claude Design 提示词 · 静室前端完全重构

> 用法：把下面「===」之间的内容整段贴给 Claude Design。它产出新前端后，交回给我接进这个 Next.js 项目部署。

===

我要为一个**中文 AI 心理咨询室**「静室」完全重构前端。这是一个 Next.js 16（App Router）+ React + TypeScript 项目，后端 API 已经写好且稳定，你只做**前端 UI/交互**。请产出干净的 React/TSX 组件 + CSS（可用 CSS 变量；项目目前用纯 CSS，不依赖 Tailwind）。

## 一句话气质
像一间**安静、温暖、被接住的房间**——不是医院，也不是玩具。专业但不冰冷，平静但不廉价。用户常常是情绪低落、读不动密集小字的人。

## 产品要做的屏 / 组件
1. **主聊天界面**（核心）：消息流（用户/AI 两种气泡）、输入框（Enter 发送、支持多行）、流式逐字显示 AI 回复、顶部品牌「静室」。
2. **虚拟陪伴者选择**（5 个，切换会改变 AI 风格）：
   - 林溪 · 暖心倾听者（人本/倾听）
   - 明远 · 认知分析师（CBT/ACT）
   - 安然 · 安稳支持师（DBT/稳定化与睡眠）
   - 若初 · 关系沟通教练（MI/边界表达）
   - 静室 · 危机稳定化（安全第一，仅在危机态出现）
3. **标准化量表入口**：PHQ-9 / GAD-7 / ISI（抑郁/焦虑/失眠自评），做成温和的分步问卷弹层，结果有解释、不下诊断。
4. **危机/安全 UI**（最高视觉优先级）：当对话出现风险信号时，界面要能突出显示热线、紧急联系、"移走危险物品"等，并明确通向**现实中的真人**。
5. **个案理解侧栏**（可选/可折叠）：展示 AI 对来访者的累积理解（主诉、触发、工作假设）。
6. **呼吸练习 / 落地练习**的小交互。
7. **会话节奏切换**：深度模式（带复核）/ 快速模式。
8. **隐私状态**：显眼地告诉用户"聊天只存在你本地""一键彻底删除"。

## 设计方向（2026，必须遵循）
- **配色**：低饱和「鼠尾草绿 / 雾蓝」为主（现产品用过 #67c2b4 系，方向对）；暖白纸感背景 `#FAF9F7` 系；深色用**真正的深**（`#0A0A0A`/`#141414`），**明暗两套都要打磨到位**（旧版深色模式有硬编码色差 bug，别重蹈覆辙）。强调色（珊瑚/暖红）**只**用在危机/求助按钮上，不要全屏花。❌ 避免高饱和、强对比、纯黑纯白硬切、医院蓝白。
- **字体**：中文思源黑体/苹方系，**字号偏大、行距松（1.6+）**；标题不要太重；情绪低落的人要能轻松读。
- **随情绪自适应**：用户情绪强度高时，界面自动**简化**——只留呼吸练习、危机按钮、联系现实支持，藏起复杂控件。
- **支持性语言，不要游戏化**：不要"连续打卡""你中断了 3 天"；改成"我一直在，需要时随时来"。
- **危机分流要显眼、要通向真人**：热线（中国大陆 12356 / 110 / 120）、紧急联系、移走危险物品——视觉最高优先级。
- **诚实**：文案与视觉都**不得**把 AI 包装成"持证治疗师/医生"——这是合规与伦理底线。
- **无障碍**：对比度达标、字号可放大、键盘可达。**移动端优先**。
- **别堆模板感**：少用一堆卡片+渐变+阴影的"AI 生成味"；留白和层级比装饰更高级。

## 参考气质
Headspace / Calm 的呼吸与留白、Finch 的温柔陪伴感、Dribbble「mental health app」精选。

## 技术约束（接回我项目时要对得上）
- 聊天通过 `POST /api/chat` 发送 `{ messages: [{role, content}], personaId?, pace?, profile?, scaleResults?, caseMap? }`，返回**流式纯文本**（逐字 append 到最后一条 AI 消息）。
- 角色 id：`linxi` / `mingyuan` / `annian` / `ruochu`；量表 id：`PHQ-9` / `GAD-7` / `ISI`。
- 双语：默认中文，可切英文（`zh` / `en`）。
- 用 `"use client"` 客户端组件 + React hooks；不要引第三方 UI 库，保持轻。

请给出：组件结构 + 完整可运行的 TSX/CSS，明暗两套主题，移动端 + 桌面两种布局。先给主聊天界面 + 角色选择 + 危机 UI 这三块，其它其次。
