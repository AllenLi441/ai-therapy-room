# 安全重构 / 危机理解 v2 —— 设计提示词

> 这是一条**独立于 P3-a/b 的工作线**,按自己的节奏推进,**上线前必须有心理专业人士评审**。
> 写给后续接手的 Claude Code(或人)。不要把它塞进 P3-b 的打磨堆里。
>
> 背景:2026-06-13,一条普通倾诉(「工作压力大,睡不好,焦虑」)被误判成自杀干预。
> 根因是隐式危机识别在 Kimi 超时的 fail-safe 里,把任何 lexicon=low 一律升级成自杀模板。
> 已用 hotfix(`628275b`)止血(见 `decideImplicitIntercept` error 分支 + 分类器中文 benign 锚点)。
> 但 hotfix 只是补丁;关键词路线本身是错的工具,这份文档定义正确的替代架构。

---

## 0. 为什么要重做(关键词路线的死穴)

纯关键词 / 词典的危险识别**两头都漏**,因为它根本不理解语义:

- **误报**:把日常词(焦虑/失眠/压力/「撑不下去」/「想消失一阵」)当成危机 → 用户被反复误判 → 对真正的安全提示**脱敏(cry wolf)**、被吓到、不再信任甚至不敢用。
- **漏报(更危险)**:真正高危的人常常**不用关键词**——「我把事情都安排好了」「不想给大家添麻烦了」「终于要解脱了」——词典一个都抓不到,但语义上是明确信号。

关键词永远在「宽到误伤」和「严到漏人」之间两难。**同一句话的意思全看上下文**(例:「撑不下去」可能是劳累,也可能是真求救)。这正是该交给会"懂"语义的模型的地方。

---

## 1. 核心原则:把「检测」和「回应」拆开

今天的耦合是:**命中关键词 → 直接换成一段写死的模板**。这同时造成了"复读、冷冰冰"和"误报吓人"两个问题。

v2 拆成两件事:

- **回应(可以完全 LLM 化、变暖、不再复读)**:由理解层生成,基于知识库,贴这个人的具体处境。
- **检测(变成一个分级信号,不是裁判)**:不再"命中即换模板",而是作为**喂给模型 + 触发安全地板**的一个分级输入。

---

## 2. 目标架构:分层 / 纵深防御

```
用户消息
   │
   ├─►  [检测层]  多个廉价+语义信号,合成一个"风险分级"(none/low/med/high + 维度)
   │        • 词典(廉价、确定性、可审计)——降级成众多输入之一,不是唯一裁判
   │        • 隐式语义分类器(LLM)——理解"安排后事/解脱/不添麻烦"这类无关键词信号
   │        • 量表结果(若有)、对话轨迹(渐进式累积)
   │
   ├─►  [理解层 / 回应]  DeepSeek 为主 + 知识库 RAG
   │        • 生成真正共情、贴合上下文的回应(替掉"那段重复文字"就在这里)
   │        • 风险分级作为 prompt 信号注入,影响语气/深度/是否做安全确认
   │
   └─►  [安全地板]  确定性、不可动摇、不依赖任何单次 LLM 调用
            • 风险真实存在时:永远能触达真人 / 永远给正确的真实资源 / 永远能退出、不被困
            • 即使 Kimi、DeepSeek 全挂,这一层照常工作
```

**一句话**:回应交给模型去暖;"高危时一定把真实求助通道摆到面前"这件事,保留确定性保证。

---

## 3. 不变式 —— 必须「确定性保证」(NON-NEGOTIABLE)

这些**不能依赖一次会超时 / 会幻觉的 LLM 调用**。用代码常量 / 规则 / 知识库,可审计、可测试:

1. **真实资源绝不让模型现编。** 热线号码(12356 / 110 / 120 等)、紧急流程,**只能从 KB / 常量取真值**。模型幻觉一个错的号码,是会害死人的错。
2. **高危时永远能触达真人。** 不论模型输出什么,真实求助通道必须被确定性地呈现。
3. **永远能退出 / 不被困。** 用户随时能离开安全模式(P0 已建,守住)。安全模式不能自我循环。
4. **地板不依赖 LLM 可用性。** Kimi/DeepSeek 超时、报错、宕机时,地板照常给出正确资源——**这正是本次 bug 的教训:fail-safe 当初依赖了 LLM 的成败来决定升级,LLM 一挂安全网就乱套。**
5. **检测的 fail-safe 要"宁缺毋滥地保守",但不能"宁可天天误报"。** 保守 ≠ 把日常词一律升级。保守的正确含义:**拿不准时交给确定性的词典判断**,而不是交给"反正升级了再说"。

---

## 4. 可以「交给模型」的部分

- **语气 / 措辞 / 共情**:暖、具体、贴人,不复读。
- **上下文贴合**:回应基于这个人说过的话,不是千人一面。
- **心理教育 / 稳定化话术的"表达"**:内容依据来自 KB(见下),但**怎么说**交给模型。
- **隐式语义检测**:理解无关键词的风险信号(安排后事 / 解脱 / 负担感 / 告别)。
- **分级而非裁决**:模型给"风险分级 + 维度",由确定性层决定地板动作。

---

## 5. 知识库(KB / RAG)要解决的三个真问题

1. **让安抚有依据、不是空话**:基于被验证过的稳定化 / 落地(grounding)技术、心理教育材料。
2. **让资源准确**:热线、分流逻辑、就近求助路径,从 KB 取真值,**绝不模型现编**。
3. **让回应贴具体处境**:检索与当前主诉相关的支持材料,而不是套同一段话。

> 注:仓库里已有 `src/lib/knowledge.ts`(`retrieveKnowledge`)和 `/api/plan` 的个案概念化,
> v2 可在其上扩展,而不是从零造。先盘点已有能力再动手。

---

## 6. 迁移策略(不要推倒重来)

- **词典不丢,降级**:从"唯一裁判"降成"众多廉价输入之一"。它仍是确定性地板的一部分(便宜、可审计)。
- **「撑不下去」这类 context-dependent idiom**:不要用改词表的方式处理。要么交给语义层,要么**单独出现时降一档当软信号、别当确诊**。
- **逐步替换模板**:先让理解层接管"回应",检测层先并行运行、对比观察,再切换裁决权。
- **保留决策日志**(`decision-log.ts` 已有):用于上线后审计误报 / 漏报。

---

## 7. ⚠️ 上线前需「专业评审」的检查点

> **我和用户都不是临床专业人士。** 给真实困境用户看的危机内容,错的代价是真实的。
> 以下每一项,上线前都要有心理 / 精神科专业人士过一遍:

- [ ] **KB 里的稳定化 / 落地话术** —— 临床上是否恰当、是否可能造成二次伤害。
- [ ] **分流逻辑** —— 什么风险等级对应什么动作(自评 / 安全确认 / 转真人 / 急救)。
- [ ] **资源清单** —— 热线号码 / 适用场景 / 是否最新有效。
- [ ] **危机回应文案** —— 语气、安全确认问法、是否符合循证危机干预原则。
- [ ] **分级阈值** —— 何时升级 / 降级;over-triage 与 desensitization 的平衡点。
- [ ] **特殊人群** —— 未成年、老年、围产期、性少数等(分类器已有 few-shot,需专业校准)。

---

## 8. LLM 自身的失败模式(为什么要保留确定性地板)

纯 LLM 不是银弹,它有自己的失败模式:**漏判、过度安抚、延迟、宕机、幻觉**。
这**不是**去掉确定性地板的理由,恰恰**是**保留它的理由:模型负责"懂"和"暖",
地板负责"无论模型怎样,真实求助通道一定在"。

---

## 9. 非目标 / 暂不做

- 不在 v2 初版做多人格(已定方向 A,单陪伴者安屿)。
- 不追求"零误报零漏报"(不可能);追求**误报不吓人、漏报有确定性地板兜底**。
- 不把诊断 / 处方能力交给模型(产品边界:AI 陪伴,非医生)。

---

## 10. 给接手者的第一步

1. 先**盘点已有能力**:`safety.ts`(词典 + 模板)、`implicit-risk.ts`(语义分类器 + 决策)、`knowledge.ts`、`/api/plan`、`decision-log.ts`。
2. 画出**当前**检测→回应的真实数据流,标出哪里依赖了 LLM 可用性。
3. 按第 3 节不变式,把"安全地板"从 LLM 调用里**解耦出来**(确定性优先)。
4. 再做"回应 LLM 化 + KB RAG"。
5. **任何危机文案 / 阈值改动,先出方案 + 找专业评审,再写码、再上线。**
