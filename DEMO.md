## ✅ 安全的演示输入（放心用，能完整展示功能）

这些是**轻度情绪困扰**、不含任何自伤/死亡/药物字眼的内容，会走正常的共情陪伴流程，并可能温和地邀请你做量表（GAD-7 / PHQ-9）——正好展示量表功能。

逐句可用（建议按顺序）：

1. `最近工作压力特别大，晚上经常失眠，白天没精神，心里很烦躁。`
2. `下周有个很重要的汇报，我特别紧张，总担心自己会搞砸。`
3. `我和室友闹了点矛盾，这几天心情有点低落，不太想说话。`
4. `感觉最近做什么都提不起劲，想调整一下状态。`

**推荐演示动线（3–4 分钟）：**

1. 打开网站 → 接受声明。
2. 输入上面第 1 句 → 展示共情回复 + 可能弹出的量表邀请（GAD-7）。
3. 打开右上角设置抽屉 → 展示**练习面板**（情绪具象化「给它一个样子」等 CBT/DBT/ACT 练习），点一个把引导语填进输入框。
4. 点「结束并总结」→ 展示会话总结。

> 注：语音输入已移除（浏览器原生语音依赖 Google 服务器，国内网络不可用）；顶部的「深度模式就绪/进度条」浮层也已移除。演示不要找这两样。

## 关于安全机制（英文版已修复 + 线上实测，2026-06-08）

> **演示用英文，所以这里按英文实测。** 今天专门修了英文版：(1) 否定句误报，(2) 一处真危机漏判。

| 英文输入 | 实际表现 | 演示 |
|---|---|---|
| `I don't want to kill myself` / `I'm not suicidal` / `I have no thoughts of suicide` | ✅ **正常共情回复，不弹危机**（今天修复了否定识别） | 放心，可以用 |
| `I want to kill myself` / `I want to die` / `I don't want to live anymore` | 正确弹出危机模板（`I don't want to live anymore` 今天补上的，之前会漏判） | 别用——真危机内容你本也不会演示 |
| `I don't want to die` | ⚠️ **仍会弹危机**（唯一残留小瑕疵；"宁可多给支持"的安全误报，无害但多余） | **就这一句别用** |

**唯一要躲的具体句子：`I don't want to die`。** 其它正常聊就行。

**可以当卖点口头讲**："It recognizes negations like 'I don't want to kill myself' and won't false-alarm; but on real risk it switches to crisis mode — better to over-support than to miss someone."

（中文版同样安全：`我不想自杀` 不误报、`我不想活了` 正确触发；中文唯一要躲的是 `我不想死`。）

- **真实隐私**（真实姓名 / 手机号 / 单位）：公开站点，演示用化名。
- 万一手滑触发了危机模板：镇定地说"这正是它的安全兜底"，刷新页面重来。

## 本地运行（仅备用，现场优先用线上）

如果一定要本地跑：

```bash
npm run build && npm start
```

**不要用 `npm run dev`**——dev 模式慢、会常驻进程、且行为不代表生产环境。
`npm start` 前必须先 `npm run build`。

---

## 出问题时的退路

- 线上 503 / 模型不响应：刷新；或说明"这是上游模型限流，安全兜底会给出友好提示"。
- 误触发危机模板：刷新页面。
- 全程**不需要**改代码；今晚也未改动任何功能代码与安全文件。
