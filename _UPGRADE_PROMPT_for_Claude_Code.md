## P1 ｜ 真实的进度 / 状态指示器（关键词：真实）
用户希望**每次生成回复时**看到"现在在做什么 + 进度"。但**必须真实**，不许假动画、假倒计时。

**现实约束（你要理解并据此设计）：** LLM 流式回复**无法精确预知"剩余秒数"**。所以：
- **状态绑定真实事件**：已发送请求 → 等待首字(TTFB) → 正在流式生成 → 完成 / 出错。这些都是前后端能真实拿到的信号。
- **进度反映真实信号**：首字到达前显示"连接中 / 思考中"，可显示**真实已等待秒数**；开始流式后，进度随**真实到达的字符/ token** 增长。若要给"预计"，只能基于历史平均长度估算并**明确标注是预计**，不得伪装成精确倒计时。
- **绝对禁止**：写死的定时器假装进度、与真实生成无关的转圈动画冒充进度。
- **出错时**（例如之前的 400）：状态要**如实显示"连接失败 / 出错"**，不许假装还在生成。
- 视觉上轻、安静，符合静室气质，不要花哨。
