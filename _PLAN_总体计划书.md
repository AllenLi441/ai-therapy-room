## 3. 目标与原则

- **先消假 → 再补漏 → 最后方向大改**（已认可 a→b→c 分层）。
- **危机相关改动绝不无人值守上线**：v2 任何危机文案/阈值改动，先方案 → 专业评审 → 再写码 → 再上线。
- **确定性安全地板不可动摇**（见 v2 文档第3节不变式）：真热线绝不模型现编、永远触达真人、永远可退出、地板不依赖 LLM 可用性。
- **诚实**：做不到/有取舍直说；验证要有可复现证据（单测/现场复测/日志），不靠"我以为修好了"。

## 5. 工作分线

**线 ①｜P3-b 体验补漏（可较快、风险低、不碰危机裁决）**
- 智能自动滚动（仅在底部跟随 + 「回到最新」）→ 顺带验证是否缓解 B1。
- 错误气泡加「重试」。
- 图片上传守卫（数量/体积上限 + vision 加载态）。
- 原生 `confirm()` → Sheet 风格确认（解 B2，并让删除可被验证）。
- 无障碍一揽子：`aria-live`、px→rem、对比度 ≥4.5:1、reduced-motion 守卫。

**线 ②｜B1 渲染问题（独立定位，不假设线①的智能滚动能解）**
- 复现 → 定位（是 state/key、还是流式结束后的 commit 时机、还是滚动容器）→ 修 → 给前后对比证据。

**线 ③｜P3-c 收尾（依赖方向已定）**
- 深化单人格安屿（pace 维度 / 示例台词 / 措辞规则 / About 展示其"声音"）。
- 清死代码 CalmMode + 修过时测试 + 合并两套量表定义（消除算分漂移）。

**线 ④｜v2 危险识别重构（独立线，慢、要评审）**
- 严格按 `_SAFETY_REARCH_v2_危机理解.md` 第10节"第一步"推进：盘点已有能力 → 画当前数据流 → 把安全地板从 LLM 调用解耦 → 回应 LLM 化 + KB/RAG → 评审 → 灰度。
